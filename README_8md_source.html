4 ## Setup
5 For this course, you need a modern C++ compiler that supports C++20. I recommend using `g++` or `clang`. `MSVC` should
6 also work, but I haven't tested it yet. Furthermore, you need `cmake` (at least version 3.14). And finally, I recommend
7 that you install one of the following libraries on your system: `libdw` (`libdw-dev`), `libdwarf` (`libdwarf-dev`).
8 These are not necessary but help producing meaningful stacktraces which makes debugging easier. I have no idea whether
9 these libraries are available for Windows...
10 
11 All the remaining dependencies are handled by cmake.

13 ## Configure the Project
14 I recommend that you use a C++ IDE like CLion or an editor like VS-Code. Both have cmake integration making your life a
15 lot easier. If you like suffering and plan on developing using a basic editor, fret naught, it's not that complicated,
16 actually. The CMake project automatically configures all targets (the executables that you will run later) for you.
17 The following targets should be configured automatically:
18 * all_tests (runs all unit tests)
19 * test_basic_structures (runs only the tests for the basic structures like Variable and Literal)
20 * test_clause (runs only the tests for the Clause class)
21 * test_solver (runs only the tests for the principal member functions of the solver class)
22 * test_unit_propagation (runs only the tests for the unit propagation)
23 
24 If you want to add other executables (e.g. a 'solve' executable that reads a problem and tries to solve it), then you
25 can add them in the main project file. For example, you could create `solve.cpp` file. In order to generate a build
26 target, you need to rerun CMake.
27 
28 If you want to add a test, then you should add `test_<name of your test>.cpp` in the `Tests`-directory. Again you need
29 to rerun CMake for the target to be generated.

31 ### CLion
32 Simply clone the project and open the folder as project with CLion. It should do everything for you. On first launch
33 CLion will probably ask you to create a CMake profile. Usually the proposed 'Debug' config is exactly what you want.
34 You can additionally create a 'Release' config by clicking on the + icon above the Debug config and it will most likely
35 create the appropriate release config automatically for you. When you're done just click ok. You can always add more
36 configs later on.
37 
38 CLion should now automatically load the CMake project and configure all targets for you. You can select which target to
39 build on the top right of your screen. All the targets mentioned above should be available.

41 ### VS-Code
42 You need to install the official Microsoft `CMake Tools` extension.
43 Clone the project and open the folder with VS-Code. If asked to trust the project, click on trust. VS-Code will probably
44 ask you to select a 'Kit' for the project. Here you select your C++-compiler. Once done, the CMake project should
45 automatically load.
46 
47 Select the `CMake Tools` extension sidebar on the left. Depending on you VSC configuration, you should see three
48 tabs (usually on the left). In the 'Project Status' tab you should see six drop-down items:
49 * Folder: We don't care
50 * Configure: Here you can select your compiler and the build configuration (Debug / Release)
51 * Build: We don't care
52 * Test: We don't care
53 * Debug: Here you can select a target and launch it with the debugger attached
54 * Launch: Here you can select a target and launch it in the console
55 
56 Under the 'Project Outline' tab you should see all libraries and executables of the project. The initial targets listed
57 above should be visible here. You can alternatively build, launch and debug from here by right-clicking on an executable
58 and selecting the desired action. Or you simply use the integrated terminal and launch the executable manually (once
59 built, they should be in you build folder).

61 ### Manually using CMake
62 If you don't want to use one of the aforementioned methods, you can manually configure the project and work with any
63 editor of your choice.
64 1. Clone the repo and change into the folder
65 2. Create a build folder for each build config (Debug / Release, e.g. build_dbg / build_rel)
66 3. To configure the project run the following command (replace <...>):
67  ```
68  cmake -S . -B <path to you build directory> -DCMAKE_BUILD_TYPE=<release/debug>
69  ```
70  CMake should run without error. Towards the end of the output you should see a list with the targets mentioned above.
71  You can manually specify the compiler using the flag `-DCMAKE_CXX_COMPILER=<the compiler of your choice>`
72 4. To build a target run the following command
73  ```
74  cmake --build <path to your build folder> -t <name of the target> -j <number of processors to use>
75  ```
76  If the build was successful, you can launch the executable directly from the terminal. For example, if you built
77  the target `test_clause` then you execute the following command:
78  ```
79  <name of your build folder>/Tests/test_clause
80  ```
81 5. Whenever you add new target (e.g. the solve executable) make sure to rerun CMake (just retype the command in 3.)
